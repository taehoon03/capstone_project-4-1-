```

│   └── README.md
```

## 빠른 시작

### 🔹 방법 1: 프론트엔드 개발 환경 (권장)

```bash
# 프론트엔드 설치 및 실행
cd frontend
npm install
npm run dev

# 브라우저에서 열기
http://localhost:5173
```

**주의**: 백엔드가 없으면 API 호출이 실패합니다.

### 🔹 방법 2: Docker Compose로 전체 실행

```bash
# 프로젝트 루트에서
docker-compose up --build

# 접속 주소
http://localhost:5173  # 프론트엔드
http://localhost:5001  # 백엔드 API
```

### 🔹 방법 3: 각각 별도로 실행

```bash
# 터미널 1: 백엔드 실행
cd test
pip install -r requirements.txt
python app.py

# 터미널 2: 프론트엔드 실행
cd frontend
npm install
npm run dev
```

## 학습 체크리스트

### SQL Injection
- [ ] 공격 구문 입력해보기: `' OR '1'='1' --`
- [ ] 모든 회원 정보가 조회되는 것 확인
- [ ] 쿼리 분석: 왜 공격이 성공했나?
- [ ] 방어 방법: 파라미터 바인딩 이해하기
- [ ] 고급 공격: `UNION` 사용해보기

### XSS
- [ ] JavaScript `alert()` 코드 입력해보기
- [ ] `<img onerror>` 태그로 공격해보기
- [ ] 쿠키 도용 시뮬레이션
- [ ] 방어 방법: 입력 이스케이프 이해
- [ ] CSP 헤더로 스크립트 차단하기

## 트러블슈팅

### Q1: "포트 5173이 이미 사용 중입니다"
```bash
npm run dev -- --port 3000
```

### Q2: CORS 에러 발생
```
Access to XMLHttpRequest at 'http://localhost:5001/...' 
blocked by CORS policy
```
**해결**: 백엔드가 실행 중인지 확인
```bash
# 백엔드 실행
cd test && python app.py

# 또는 Docker
docker-compose up
```

### Q3: "find: command not found" 에러 (Windows)
Windows PowerShell에서는 Unix 명령어를 지원하지 않습니다.
대신 PowerShell 명령어를 사용하세요.

### Q4: Python 모듈을 찾을 수 없음
```bash
pip install -r requirements.txt
```
